# Claude Code 路由表

## ⚠️ 給 Claude Code 的工作規則(專案擁有者非工程師,正在學習)

請在這個專案裡一律遵守以下規則,直到使用者另外說明為止:

1. **一次只做一個小任務**。例如「先寫好 1-直播素材準備 底下的 format_detection.js」,不要因為使用者描述了整個流程就一次把 5 個 skill 全部生出來。
2. **做完一小塊就停下來,等使用者確認**,不要連續產生大量檔案。先列出你打算建立/修改的檔案清單,讓使用者看過再動手。
3. **每完成一個能獨立運作、驗證過的小功能,主動提醒使用者**:「這個可以先存檔了,去左側『原始檔控制』打 commit 訊息、按 ✓、再按同步推送上 GitHub」。不要等到整個 skill 都做完才提醒存檔。
4. **任何新寫的腳本都要實際執行測試過一次,並把測試結果或錯誤訊息給使用者看**,不要只生成程式碼就說完成了。
5. **解釋用簡單的中文**,假設使用者沒寫過 code、剛開始學;遇到術語(例如 ffmpeg、whisper、JSON)請順手用一句話解釋。
6. **開始任何看起來會處理很多檔案或很長對話的任務之前**,提醒使用者先打 `/usage` 看一下額度還剩多少;如果額度偏低,建議先完成手上這一小步、commit 存檔,再考慮要不要繼續。
7. 完成一個小任務後,可以建議使用者打 `/clear`(如果這個任務已經做完,接下來要做全新的事)或 `/compact`(如果還要接續做、但對話已經很長),保持對話輕量,避免額度被「越講越長的對話」吃掉。

## 5 個 Skill 清單

### 1️⃣ 直播素材準備
`misea-videocut-skills:直播素材準備`

功能：準備直播源或錄屏素材
- 提取音頻
- 檢測視頻格式
- 預處理和轉碼

### 2️⃣ 智能分析和剪輯建議  
`misea-videocut-skills:智能分析`

功能：分析視頻並生成剪輯建議
- 轉錄（中文、台語、英文）
- 靜音檢測
- 精彩時刻檢測（搞笑、驚喜）
- 場景切換檢測
- 風格分析

### 3️⃣ 審核和預覽
`misea-videocut-skills:審核預覽`

功能：交互式預覽和確認
- 生成分鏡頁面
- 時間線預覽
- 啟動審核服務器

### 4️⃣ 多格式輸出
`misea-videocut-skills:多格式輸出`

功能：生成最終成片
- YouTube Shorts（9:16）
- 完整版（16:9）
- 自動字幕
- 音樂匹配

### 5️⃣ 風格學習和自進化
`misea-videocut-skills:風格學習`

功能：學習火紅視頻風格
- 分析視頻 URL
- 提取特徵
- 生成新規則
- 自動優化
